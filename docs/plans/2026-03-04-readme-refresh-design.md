# README 重写设计

## 目标

把仓库首页改成一份可直接面向 GitHub 访问者阅读的中文 README，替换现有乱码内容，并准确反映当前项目已经实现的功能、运行方式和验证方式。

## 读者

- 第一次打开仓库的玩家或同学
- 想快速本地运行项目的开发者
- 想判断当前功能边界和工程状态的协作者

## 方案

采用“产品型 README”结构，优先回答三个问题：

1. 这是什么项目
2. 现在已经能做什么
3. 我怎么在本地跑起来并验证

文案以中文为主，不做中英双语并列。内容会避免写死未经验证的结论，只写当前代码和测试可以支撑的信息。

## 信息组织

README 将按以下顺序展开：

1. 项目简介
2. 当前已实现能力
3. 游戏流程与特色规则
4. 快速开始与常用命令
5. 测试与验证
6. 项目结构
7. 相关文档与后续方向

## 边界

- 不在 README 中复制完整掼蛋规则手册
- 不承诺尚未实现的线上部署、账号体系或持久化能力
- 不把本地工具配置、代理配置或个人工作流写进 README

## 成功标准

- README 不再出现乱码
- 启动地址、命令和测试命令与当前仓库一致
- 能体现最近完成的房间 UI、房主重连、手动进贡/还贡等更新
